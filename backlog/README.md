# Backlog

This directory keeps the project-local backlog for `splendor_dual_pretzel`.

Priority scale:
- `P1` unlocks the core objective from `README.md`: a mobile-friendly web-based digital version of Splendor Dual.
- `P2` expands the project into a complete playable experience or a strong player-facing demo.
- `P3` hardens, preserves, and polishes the experience.

Working files:
- `backlog/board.md` is the source-of-truth ranked backlog.
- `backlog/tickets/` contains one markdown file per ticket.
- `backlog/triage/` stores the current triage summary and any history snapshots.
- `backlog/activity-logs/` stores one short ongoing activity log per automation.
- `backlog/reports/` stores detailed automation run reports.
- `backlog/product-manager-feedback.txt` stores question-and-answer prompts for human product feedback.
- `backlog/product-manager/` stores the latest Product Manager summary and screenshot outputs for review.
- `backlog/collaboration-loop.md` describes how Product Manager, Developer, and Tester should hand work to each other.
